Mini NPU Simulator
본 프로젝트는 NPU(Neural Processing Unit)의 핵심 연산인 MAC(Multiply-Accumulate)을 시뮬레이션하고, 패턴 판별 로직 및 데이터 크기에 따른 성능 변화를 분석하는 파이썬 콘솔 애플리케이션입니다.

1. 실행 방법
환경 준비: Python 3.6 이상의 버전이 설치되어 있어야 합니다. (외부 라이브러리 설치 필요 없음)

데이터 준비: data.json 파일이 main.py와 동일한 디렉토리에 위치해야 모드 2가 정상 작동합니다.

프로그램 실행: 터미널에서 아래 명령어를 입력합니다.

Bash
python main.py
모드 선택:

1번(사용자 입력 모드): 콘솔에 직접 3x3 행렬 데이터를 입력하여 판정합니다. (행별로 공백 구분 입력)

2번(data.json 분석 모드): 준비된 JSON 파일을 로드하여 5x5, 13x13, 25x25 등 다양한 크기를 일괄 테스트합니다.

2. 구현 요약
라벨 정규화(Normalization): 입력 데이터의 expected 값(+, x)이나 필터 키(cross, x)를 프로그램 내부 표준 라벨인 Cross와 X로 변환하여 처리합니다. 대소문자 구분 없이 문자열을 처리하여 데이터 입력의 유연성을 확보했습니다.

MAC 연산 구현: NumPy와 같은 외부 라이브러리 없이 순수 Python의 이중 반복문을 사용합니다. 패턴 행렬과 필터 행렬의 동일한 인덱스(i,j) 위치에 있는 값을 곱하고, 이를 전역 변수에 누적 합산하여 최종 점수를 산출합니다.

동점 처리 정책(Epsilon): 부동소수점 연산 특성상 발생하는 미세한 오차를 해결하기 위해 EPSILON = 1e-9를 적용했습니다. 두 필터 점수의 차이(abs(score_a - score_b))가 이 값보다 작을 경우 UNDECIDED(판정 불가)로 분류하여 판정의 신뢰도를 높였습니다.

3. 결과 리포트
3.1 FAIL 케이스 원인 분석

모드 2 실행 시 발생하는 FAIL 케이스의 주된 원인은 데이터 규격 불일치입니다.

의도적 오류 검증: data.json 내에 포함된 size_5_02_error_test 케이스는 5x5 필터를 사용하도록 설정되어 있으나, 실제 입력 패턴은 4x5 행렬로 구성되어 있습니다.

처리 결과: 프로그램은 연산 전 행렬의 행과 열 개수를 검증하며, 불일치가 발견되면 IndexError로 프로그램을 중단시키는 대신 해당 케이스를 FAIL로 처리하고 사유(패턴 크기 불일치)를 리포트에 기록합니다. 이를 통해 실제 하드웨어 가속기에서 발생할 수 있는 데이터 정렬(Alignment) 오류나 규격 미달 상황을 안전하게 시뮬레이션했습니다.

4. 구조 요약 (Class Design)
NPUSimulator: 순수 반복문 기반 MAC 연산 및 Epsilon 기반 비교 로직 담당.

DataManager: 콘솔 입력 유효성 검증, JSON 로드, 라벨 표준화 담당.

AppController: 전체 실행 흐름 제어 및 성능 리포트 출력 담당.
